# OctoAcme Project Management Docs

This folder is the landing page for OctoAcme’s project management documentation. It provides a single entry point to the team’s shared process guidance and helps centralize institutional knowledge for day-to-day delivery and use in Copilot Spaces.

OctoAcme’s process follows a lightweight but complete lifecycle: initiation, planning, execution, release, and retrospective improvement. Projects begin by clarifying business needs, measurable goals, stakeholders, milestones, and early risks. Planning then breaks work into shippable increments, prioritizes backlog items, estimates effort, and aligns delivery with acceptance criteria and a Definition of Done.

Roles are clearly defined across the lifecycle: Project Managers coordinate delivery and communication, Product Managers own outcomes and prioritization, developers implement and test changes, QA validates quality and acceptance criteria, and stakeholders provide input and approvals. Execution is run through visible board stages and recurring team cadences, with regular risk review, escalation paths, and status communication.

Quality is built in throughout delivery through automated and manual testing, CI/security checks, review standards for pull requests, and release-readiness verification. After release, retrospectives and tracked action items turn lessons learned into continuous process improvement.
